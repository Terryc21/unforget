# tests/ — unforget self-test corpus

Regression bench for the deterministic helpers under `scripts/`. Each helper
produces JSON; the harness runs the helper against a frozen fixture project,
normalizes the output, and diffs it against a golden file in `tests/golden/`.

Motivated by the v0.2 Phase 4 retest, which caught three real bugs (T-8 path
encoding, T-12 audit-filename pattern, T-13 per-finding-ID extraction) only
because the author hand-walked Stuffolio. This corpus is the bench that would
have caught them on commit, not on retest.

## Run

```bash
bash tests/run.sh
```

Exits 0 with `PASS:` per helper when goldens match. Exits 1 with a unified
diff when any helper diverges.

Requires only `bash` and `python3` (3.9+). No pytest, no third-party
dependencies.

## Coverage matrix

| Helper | Golden | Notes |
|---|---|---|
| `scripts/scan_surfaces.py` | `golden/scan_surfaces.json` | Run with `--include-comments`. Surface 5 reports `not-a-git-repo` (fixture has no `.git/`). Surface 6 candidates are cleared + notes suppressed during normalization (the helper resolves `~/.claude/projects/<encoded>/memory/` from the runner's cwd, which is not stable across machines). |
| `scripts/check_format_version.py` | `golden/check_format_version.json` | Run against the fixture's `UNFORGET.md`. Confirms the v1 marker + four section headers. |
| `scripts/encode_project_path.py` | `golden/encode_project_path.json` | Run against a fixed string (`/Volumes/2 TB Drive/Coding/GitHub/unforget-test`) so the golden does not depend on the runner's filesystem layout. Catches the T-8 rule directly. |
| `scripts/dedup_findings.py` | `golden/dedup_findings.json` | Run against `fixtures/dedup-input.json`. Confirms the Jaccard fuzzy-merge clusters two near-duplicate headlines and keeps two distinct findings separate. |

### Not covered (deferred)

- **`scripts/prune_backups.py`** — destructive (deletes files). Belongs in its
  own test that creates and tears down a temp dir. Out of scope for the
  initial corpus.
- **Surface 6 (memory files)** — requires either a `--memory-dir` override
  flag in `scan_surfaces.py` or a mock `~/.claude/projects/` tree in the
  fixture. Neither exists today; the normalizer suppresses Surface 6 output
  so the bench remains stable. Add this when the override flag lands.
- **Surface 5 (GitHub issues)** — the helper never invokes `gh`; it only
  reads `.git/config`. The fixture is intentionally not a git repo, so this
  surface returns `not-a-git-repo`. A follow-up could plant a stub `.git/`
  with a github.com origin to exercise the `github-detected-defer-to-gh-cli`
  branch.

## Re-baselining

When a helper's behavior changes intentionally (new heuristic, format
update, etc.), regenerate the goldens:

```bash
bash tests/run.sh --bless
git diff tests/golden/   # review the change
```

Only commit golden updates after reviewing the diff. The whole point of the
bench is that goldens describe *intended* behavior — re-blessing without
review re-introduces the very class of bug the bench prevents.

## Why JSON, not the markdown candidate report

The Phase 2 candidate report shown to users is markdown prose generated by
the LLM from `scan_surfaces.py`'s JSON output. The markdown layer is
non-deterministic (LLM rendering varies session-to-session) and would
brittle-couple the bench to SKILL.md prose. Diffing the helper's JSON
exercises every algorithm a regression could break — surface heuristics,
filename patterns, regexes, exclusions — without depending on prose
that's allowed to change.
